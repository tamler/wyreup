## Drift found and fixed

1. **Root README**: `image-to-pdf → merge-pdf` CLI example was redundant — `merge-pdf` requires 2+ PDFs but receives only one from the previous step. Updated to `image-to-pdf` (single command) + chain example with `pdf-compress`.
2. **Root README**: version `v0.1.0` in header. Updated to `v0.5.0`.
3. **Root README**: agent skill row referenced npm packages (`@wyreup/skill` etc.) which are superseded by `wyreup install-skill`. Updated.
4. **core/README.md**: `runTool` function advertised but never existed. Replaced with the real API: `createDefaultRegistry()` + `tool.run()`.
5. **skill/skill.md**: `wyreup split-pdf doc.pdf --output-dir ./pages/` used wrong flag name. The CLI flag is `-O`, not `--output-dir`. Fixed.
6. **All CLI examples**: all `wyreup <tool>` examples were previously broken (CLI only had `list`, `init-tool`, `install-skill`). Now fixed by shipping `run`, shorthand routing, and `chain`.
